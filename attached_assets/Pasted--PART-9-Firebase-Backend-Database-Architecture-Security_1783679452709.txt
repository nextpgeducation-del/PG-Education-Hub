# PART – 9 : Firebase Backend, Database Architecture, Security, PWA & Deployment

## Backend Architecture

Build the entire website using **Firebase** as the backend.

Use:

* Firebase Authentication
* Cloud Firestore Database
* Firebase Storage
* Firebase Hosting (optional)
* Firebase Cloud Messaging (Notifications)

The project must work without any placeholder errors.

# Security Rules

Students:

* Read only their own data.
* Update only allowed profile fields.
* Cannot access Admin data.
* Cannot modify fees.

Admin:

* Full access to all collections.

Public:

* Read-only access to public website pages.
* No direct database write access.
