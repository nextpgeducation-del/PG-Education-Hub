# PART – 7 : Student Fee Management System (Admin & Student)

## Student Fee Management System

Create a complete **Fee Management System** integrated with Firebase Firestore.

The system must automatically create a fee record for every new student.

---

# Automatic Fee Dashboard

Immediately after a student registers or is added by the Admin:

Automatically create a **12-month fee schedule** starting from the student's **Admission Month**.

### Example

If Admission Date is:

**July 2026**

Automatically generate:

* July 2026
* August 2026
* September 2026
* October 2026
* November 2026
* December 2026
* January 2027
* February 2027
* March 2027
* April 2027
* May 2027
* June 2027

No manual month creation should be required.

---

# Monthly Fee Structure

For every month store:

* Month Name
* Year
* Monthly Fee Amount
* Discount (Optional)
* Late Fee (Optional)
* Amount Paid
* Remaining Balance
* Payment Status
* Payment Date
* Receipt Number
* Payment Mode
* Remarks

---

# Payment Status

Available Status:

🟢 Paid

🟡 Partially Paid

🔴 Pending

Status should automatically update based on payment amount.

# Security Rules

* Only Admin can modify fee records.
* Students can only view their own fee information.
* Every fee transaction must be saved permanently in Firestore.
* Refreshing or logging out must not remove payment history.
* Maintain an audit log of all fee updates (date, time, admin, action).

* Compatible with the Student Dashboard and Admin Dashboard
* Designed using reusable components and modern UI/UX principles.
