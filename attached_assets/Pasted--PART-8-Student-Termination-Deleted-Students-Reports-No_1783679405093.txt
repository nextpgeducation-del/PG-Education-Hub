# PART – 8 : Student Termination, Deleted Students, Reports, Notifications & Backup System

## Student Termination System

Create a professional **Student Termination Management System**.

The Admin should never permanently delete a student immediately.

Instead, students should first be moved to a **Terminated Students** section.

---

# Terminate Student Button

Every student row inside **Registered Students** must contain:

🚫 **Terminate Student**

When clicked:

Show confirmation popup.

Popup Message:

> Are you sure you want to terminate this student?
>
> This student account will become inactive immediately.
>
> The student's data will remain safely stored for **60 days**.
>
> During these 60 days, the Admin can restore the account at any time.
>
> After 60 days, the account should be permanently deleted automatically.

Buttons

* Cancel
* Confirm Termination

---

# After Confirmation

Immediately perform these actions:

* Change Student Status to **Terminated**
* Disable Student Login
* Remove student from Active Students list
* Move the student to **Terminated Students**
* Save Termination Date
* Start a 60-day countdown
* Keep all student records (profile, fees, notes, results, attendance if added later)

Do **not** permanently delete the student at this stage.

---

# Terminated Students Page

Create a dedicated page inside Admin Dashboard.

Display:

* Student ID
* Admission Number
* Student Name
* Father's Name
* Mobile Number
* School Name
* Class
* Admission Date
* Registration Date
* Termination Date
* Days Remaining
* Termination Reason
* Current Status

---

# Countdown Timer

Display remaining days.

Example:

59 Days Remaining

45 Days Remaining

10 Days Remaining

1 Day Remaining

0 Days Remaining

Countdown should calculate automatically from the Termination Date.

---

# Restore Student

Provide a **Restore Student** button.

When clicked:

* Move student back to Registered Students
* Enable Student Login
* Restore previous status
* Restore all fee records
* Restore dashboard data
* Restore notes access
* Restore mock test history

Nothing should be lost.

---

# Permanent Delete

Provide a **Permanently Delete** button.

Show confirmation popup.

Message:

> This action cannot be undone.
>
> Do you want to permanently delete this student?

Buttons:

* Cancel
* Delete Permanently

When confirmed:

Delete:

* Student Profile
* Authentication Record
* Fee History
* Dashboard Data
* Notifications
* Activity History

Only after explicit confirmation.

---

# Automatic Deletion

If a student remains in the **Terminated Students** section for 60 days:

Automatically:

* Permanently delete the student record
* Delete authentication account
* Remove related files from storage (if applicable)
* Remove student from search results

This process should run automatically (scheduled cleanup/background function).

---

# Security

* Only Admin can terminate, restore or permanently delete students.
* Students cannot access the Terminated Students module.
* Protect all operations using Firebase Authentication, Firestore Security Rules and role-based access control.
* Keep deleted records secure until the 60-day retention period expires.

Generate this module as production-ready, responsive and fully integrated with the Admin Dashboard, Student Dashboard and Firebase backend.
